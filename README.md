# Система поиска и сравнения рентгеновских снимков (OPTG)

Веб-приложение для поиска и сравнения ортопантомограмм (ОПТГ) с использованием нейронных сетей. Система позволяет загружать рентгеновские снимки, сравнивать их между собой и искать похожие изображения в базе данных.

## Возможности

- **Сравнение изображений**: Загрузка двух рентгеновских снимков и вычисление степени их схожести
- **Поиск по базе данных**: Поиск наиболее похожих снимков из базы данных PostgreSQL
- **Извлечение признаков**: Автоматическое извлечение признаков из изображений с помощью нейронной сети
- **Многоязычный интерфейс**: Поддержка русского и английского языков
- **Асинхронная обработка**: Быстрая параллельная обработка нескольких изображений

## Технологии

- **Backend**: FastAPI (Python)
- **Нейронная сеть**: ConvNeXt Tiny (через библиотеку timm)
- **База данных**: PostgreSQL с асинхронным доступом (SQLAlchemy + asyncpg)
- **Обработка изображений**: OpenCV, Albumentations
- **Frontend**: HTML/CSS/JavaScript с шаблонами Jinja2

## Установка

### Предварительные требования

- Python 3.8 или выше
- PostgreSQL база данных
- CUDA-совместимая видеокарта (опционально, для ускорения на GPU)

### Шаги установки

1. **Клонирование репозитория**
   ```bash
   git clone <repository-url>
   cd FinalProject
   ```

2. **Создание виртуального окружения**
   ```bash
   python -m venv .venv
   # Windows:
   .venv\Scripts\activate
   # Linux/Mac:
   source .venv/bin/activate
   ```

3. **Установка зависимостей**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройка базы данных**

   Создайте базу данных PostgreSQL.

   Настройте файл `backend/config.env` для параметров приложения:
   ```
   HOST=127.0.0.1
   PORT=8000
   ```

   Создайте файл `backend/Databases/db_config.env` с настройками подключения к БД:
   ```
   SQLALCHEMY_DATABASE_URL=postgresql+asyncpg://username:password@localhost:5432/database_name
   ```
   Файл `db_config.env` используется скриптом `Database.py` для подключения к базе данных.

5. **Подготовка модели нейронной сети**

   Поместите обученную модель в папку `backend/NNModels/PretrainedModels/` с именем `final_model.pth`.
   Модель должна быть обучена с использованием архитектуры ConvNeXt Tiny.

6. **Создание таблиц в базе данных**

   Запустите приложение, таблицы создадутся автоматически при первом запуске (если настроен auto-create).

## Использование

### Запуск приложения

```bash
cd backend
python main.py
```

Приложение будет доступно по адресу: http://127.0.0.1:8000

### Основные страницы

- **Главная страница** (`/`): Информация о системе
- **Загрузка** (`/upload/`): Загрузка изображений для сравнения или поиска
- **Сравнение** (`/compare/`): Сравнение двух загруженных изображений
- **Результаты** (`/result/`): Просмотр результатов сравнения или поиска

### API эндпоинты

#### POST `/search/`
Поиск похожих изображений в базе данных.

**Параметры:**
- `user_image` (UploadFile): Изображение для поиска

**Поддерживаемые форматы:** `.png`, `.jpg`, `.jpeg`

#### POST `/compare/`
Сравнение двух загруженных изображений.

**Параметры:**
- `user_image` (UploadFile): Первое изображение
- `user_image1` (UploadFile): Второе изображение

**Поддерживаемые форматы:** `.png`, `.jpg`, `.jpeg`

### Пример использования

1. Перейдите на страницу "Загрузка"
2. Выберите режим: "Сравнение" или "Поиск в базе"
3. Загрузите одно или два изображения
4. Нажмите "Анализировать"
5. Просмотрите результаты со степенью схожести (в процентах)

## Архитектура нейронной сети

Система использует модель **StackedChannelsModel** на базе архитектуры **ConvNeXt Tiny**:

- Вход: два изображения, объединенные по каналам (6 каналов: 3 для первого + 3 для второго)
- Backbone: ConvNeXt Tiny (предобученная на ImageNet)
- Классификатор: Global Average Pooling → Linear(256) → ReLU → Dropout → Linear(2)
- Выход: вероятность того, что изображения принадлежат одному человеку

## Утилиты

### Создание датасета (`utils/create_dataset.py`)
Скрипт для формирования CSV-файла с метаданными изображений из папок с структурой `path/age/sex`.

### Предобработка изображений (`utils/preprocessing_image.py`)
Содержит трансформации для:
- Обучения (сильные аугментации)
- Валидации (слабые аугментации)
- Тестов (без аугментаций)

Доступные датасеты:
- `SmiCLRDataset` - для contrastive learning
- `SmiMIMDataset` - для masked image modeling
- `MultiTaskDataset` - для многозадачного обучения (ID, возраст, пол)
- `PairDataset` - для обучения на парах изображений
